02 · Before installing

Run Hermes in a sandbox, not your main machine.

Option A · Mac mini

Best if you have a dedicated machine at home or the office. Create a separate macOS user, SSH into it, and keep Hermes files isolated from your personal laptop.

$ ssh your-user@macmini.local
$ mkdir -p ~/hermes-workshop && cd ~/hermes-workshop

Option B · Hostinger VPS

Best for the workshop: a clean Ubuntu VPS, easy to reset, accessible from anywhere, and safe to use for terminal/file/browser tools.

hostinger.com?REFERRALCODE=0WRPRECHT01I
$ ssh root@YOUR_SERVER_IP
# apt update && apt upgrade -y

Minimum checklist

Ubuntu 22.04+ or macOS, SSH access, sudo/admin rights, 2GB+ RAM, stable internet.

Create a clean shell

Use tmux so the install keeps running even if the connection drops.

$ tmux new -s hermes

Security habit

Use a new folder/user, do not paste private production secrets during the workshop, and prefer temporary API keys.

Hands-on ruleFirst choose the sandbox. Then copy/paste the install command.
03 · Workshop setup flow

Start simple: install, pick a model, enable tools, connect channels.

$ curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
$ hermes setup # guided config
$ hermes model # choose provider/model
$ hermes doctor # verify health

$ hermes tools # enable toolsets
$ hermes gateway setup # WhatsApp, Telegram, etc.

Configuration lives in predictable places

~/.hermes/config.yaml for behavior, ~/.hermes/.env for secrets, ~/.hermes/skills/ for procedures, ~/.hermes/sessions/ for transcripts.

The workshop should emphasize “configure once, reuse everywhere.”

05 · Model setup

Use any model — but for this workshop, I recommend OpenAI Codex via OAuth.

Why this path

No API key pasted into the terminal, credentials are stored in ~/.hermes/auth.json, and it is usually the most efficient default for hands-on agent work.

If you already use another provider, Hermes can switch later with hermes model.

$ hermes login --provider openai-codex
# Open the URL, enter the device code, approve access

$ hermes model
# Choose “OpenAI Codex” and pick the recommended Codex model

$ hermes doctor
$ hermes chat -q "Say hello from my new Hermes agent"

Step 1

Run the OAuth login command and complete the browser/device-code flow.

Step 2

Run hermes model, select OpenAI Codex, then accept the suggested model.

Step 3

Run a one-line chat test before adding tools, skills, or channels.

Auth noteDirect OpenAI API keys also work, but OAuth is the clean workshop path.

07 · Skills

Skills are reusable procedures — the agent’s operating manual.

Instead of re-explaining how you work every time, encode the workflow once and let Hermes load it when relevant.

hermes-agentnotionbernardo-voicebeprecht-content-chiefgoogle-ads-api-operationslocal-service-business-operationspowerpointclaude-designgithub-pr-workflowsystematic-debugging

How to explain it in the workshop

  • Skills are markdown files with trigger conditions, steps, commands, pitfalls, and verification.
  • They are loaded into context only when useful.
  • They capture “how Bernardo/Hermes does this,” not generic advice.
  • Good skills turn successful experiments into durable leverage.

08 · Memory + sessions

Two different forms of continuity

Persistent memory

Compact durable facts: user preferences, stable environment details, recurring conventions.

Example: “Bernardo prefers concise WhatsApp replies.”

Session search

Retrieval over past conversations: useful for “where did we leave off?”, decisions, debugging context, and prior artifacts.

Example: “Find the briefing workflow session.”
Workflow: capture durable preference → save memory → later task loads it automatically
Workflow: need past details → session_search → reconstruct context → continue without asking user to repeat
Important distinctionMemory is durable facts; session search is history retrieval

13 · Architecture pattern

The pattern: LLM for reasoning, deterministic workflows for execution.

Reasoning

Summarize, classify, decide next steps, generate drafts, review anomalies.

Execution

APIs, scripts, databases, file writes, commands, Notion updates, scheduled jobs.

Governance

Human approval for risky actions, secret handling, client isolation, verification before reporting success.

Why this matters

It avoids the “AI magic box” trap. Hermes is powerful because it combines judgment, tools, context, and repeatable procedures with visible evidence.
